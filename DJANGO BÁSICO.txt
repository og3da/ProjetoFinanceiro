DJANGO EXPLICANDO O BÁASICO

- APP
UM APP É UMA SEÇÃO NO DJANGO. CADA PÁGINA PODE SER UM APP NOVO, CADA APP TEM UM SIGNIFICADO ESPECÍFICO. NO NOSSO PROJETO POR EXEMPLO A "HOME" É UM APP.

- VIEW
UMA VIEW CONTÉM FUNÇÕES QUE RECEBEM OS REQUESTS HTTP E RETORNAM UM RESPONSE, COMO DOCUMENTOS HTML (EX: APP MAIN, A VIEW DELE RETORNA A PAGINA HOME)

- URL
É UM ARQUIVO QUE USAMOS PARA DIRECIONAR À UMA FUNÇÃO CONTIDA EM "VIEWS". CADA APP TEM SEU urls.py, DEVEMOS TAMBÉM FAZER O ROTEAMENTO DO PATH NO ARQUIVO urls.py DA PASTA RAIZ

- TEMPLATES
É A PASTA QUE CRIAMOS PARA ARMAZENAR OS ARQUIVOS HTML DO DETERMINADO APP
